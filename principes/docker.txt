Предоставление благодаря docker exec контейнера php в виде:
	docker exec -i --user=1000:1000 docker-server  php "$@"
позволяет работать таким VSCODE расширениям как INTELLISENSE, но не позволяют запускать
такие исполняемые файлы COMPOSER, как phpcs

Решением является:
	- создание томов с локальным COMPOSER 
	- создание исполняемого файла /usr/local/bin/phpcs в OS, который ссылается на контейнер,
		предоставляюю путь к внешнему исполняемому файлу phpcs
			docker exec -i --user=1000:1000 docker-server ./vendor/bin/phpcs "$@"
	- ссылка на исполняемый файл из внешнего тома, которая передаётся в контейнер PHP,
		с предустоновленным COMPOSER обрабатывается и возвращается результат в расширение IDE
